# Blueprint: the statement-audit website

A [rocqblueprint](https://github.com/reiniscirpons/rocqblueprint) site
giving result-first access to the library's formal statements for
mathematicians with no Rocq background: each result page shows the
verbatim formal statement, a plain-mathematics decoding, and
`uses`-links to every definition the statement depends on (the
Dictionary), plus a clickable dependency graph. **No proofs are
shown** — the audit contract is described in `docs/PLAN_WEB.md`.

Published site (deployed by `.github/workflows/blueprint.yml`):

> https://llm4rocq.github.io/digraph-theory/

## Layout

```
blueprint/src/
  web.tex / print.tex     entry points (HTML / PDF)
  plastex.cfg             plasTeX renderer config
  macros/                 common + per-target macros
  content.tex             chapter list
  bp_intro.tex            hand-written landing chapter
  bp_dictionary.tex       GENERATED — definition entries
  bp_unified.tex,
  bp_families.tex,
  bp_paths.tex,
  bp_general.tex          GENERATED — one theorem block per result
```

The `bp_*` chapters (except `bp_intro`) are generated by
`scripts/gen_blueprint.py` from the canonical content in
`docs/web/chapters/` (shared with the PDF companion
`docs/formal/digraph_formal.pdf`), the statement closures
(`docs/web/closure.json`) and the extracted verbatim quotes
(`docs/web/quotes/*.code`). **Edit the PDF chapters and regenerate;
never edit `bp_*` files by hand.** The `\uses{...}` edges come from
the machine-computed closures, so the dependency graph cannot drift
from the audited content; CI re-derives everything from the compiled
library and fails on any difference.

## Build locally

```bash
pip install rocqblueprint        # needs graphviz + headers
python3 scripts/extract_quotes.py
python3 scripts/gen_blueprint.py
cd blueprint/src
rocqblueprint web                # -> blueprint/src/web/
rocqblueprint serve              # local server
```
